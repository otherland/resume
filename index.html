2023 — Present Enterprise SaaS

Canary Care

Modernizing a connected care platform used by 200+ UK facilities to monitor 10,000+ elderly residents

When I joined Canary Care, the codebase had grown over nearly a decade of product evolution. The frontend was a hybrid of legacy PHP-rendered views layered with progressively-enhanced JavaScript. The backend was a patchwork of PHP microservices, Python services added over time, and infrastructure tightly coupled to Rackspace VMs with manual deployments. This is a health and safety company — if the systems go down, care staff can't monitor vulnerable residents. Reliability matters here more than anywhere I've worked.

I led the frontend modernization: migrated the build system from Webpack 4 to Vite (hot module reload went from 30+ seconds to near-instant), implemented Yarn Berry with Plug'n'Play, and set up containerized deployments. The React migration reached Phase 9 — I documented and built 50+ endpoints across nine phases, from authentication and system status through to user management and hub configuration. The Python FastAPI backend I wrote for these delivered 3–72% performance improvements over legacy PHP, with reference data endpoints showing 70%+ speedup.

Containerization was the bigger job. I audited the system and found 18 distinct services (not the 26 everyone assumed), mapped every dependency, and laid out a phased plan. I wrote Dockerfiles with multi-stage builds for production, a consolidated Azure DevOps pipeline building all images in parallel with multi-stage approval gates, and Terraform configs for ECS task definitions with proper IAM role separation. By early 2026, all services were containerized and running in production on ECS.

I also led the response to a security incident where passwords were being logged unencrypted in Grafana. Within 30 minutes I'd traced it to a specific PHP file, implemented a Grafana Alloy filter as immediate mitigation, then coordinated a multi-layer fix across teams: Grafana filter, Apache filter, and PHP-level redaction. Separately, I drove a migration of hardcoded credentials to AWS Systems Manager Parameter Store — 64% migrated in the first month, database credentials at 100%.

Beyond the code: Grafana dashboards for service version tracking and deployment history, 677 Confluence pages consolidated into searchable knowledge bases, and a tracking repo with 26 active operational scripts.

2024 — Present Founder / Startup

Wexley Capital

Built the entire technical stack for a B2B lead generation and sales automation platform from scratch — wexleycapital.com

Wexley Capital is a B2B automation and lead generation platform I built from scratch. Registered as both a US LLC and UK Ltd, currently in an advisory incubator programme. The tech stack processes millions of contacts across multiple data sources, runs advanced browser automation, and combines AI-powered lead scoring with email verification at scale.

At the centre is a unified BigQuery dataset of 162M+ deduplicated contacts from 7 sources — Pitchbook, Crunchbase, LinkedIn, Apollo, and others. I built the ETL pipelines, deduplication logic, and query infrastructure to make this data usable. On top of that, the Lead Scorer uses a multi-model AI system with API key rotation across Gemini instances, scoring hundreds of thousands of records with ThreadPoolExecutor for parallel throughput of 100–200 ops per minute.

The Apollo Scraper is the most technically involved piece — a three-tier extraction system. Primary tier uses direct HTTP requests, reconstructing exact API request patterns including cryptographic transaction IDs and client hints, getting 10–20x the performance of browser automation. Fallback tier uses Playwright with injected JavaScript for tab visibility spoofing and DOM scraping. The orchestration tier coordinates parallel scrapes across multiple authenticated accounts with per-account locking, LRU selection, and automatic session refresh.

LinkZen is a full API product with a PostgreSQL backend (6.8M exec records, 2.5M companies), rate limiting, API key management, and audit logging. It combines domain MX checking, team page crawling, LinkedIn public API scraping, and email verification via direct SMTP for Google/Microsoft and third-party APIs for other providers. A CSV folder watcher daemon monitors for new files, verifies addresses, and streams results to the database with progress tracking.

XEngine automates Twitter/X engagement with an HTTP API client that bypasses browser overhead entirely, reconstructing Twitter's exact request patterns. Runs as a Unix-style daemon via launchd with SQLite persistence, token bucket rate limiting, and persona-driven content generation using Claude for voice-matched tweets and contextual replies. The Sales Assistant is a client-side SPA with 130+ categorized objection handlers and URL-routable content for structured discovery and objection-handling.

The full platform connects data sourcing, AI scoring, email verification, outreach automation, and sales enablement into one pipeline. Each piece solves a specific problem in the B2B sales workflow.

Infrastructure & Security

Enterprise email systems, Microsoft 365 automation, vulnerability research, and data pipelines — wexleylabs.com

I built an email campaign platform managing 1,500+ sending accounts with a Django/Celery/Redis backend. The system handles sender reputation tracking, bounce processing, and field-level encryption using Fernet for credential storage. Running that many accounts means understanding DNS (SPF, DKIM, DMARC), IP warming, and domain reputation at an operational level.

For Microsoft 365, I automated tenant provisioning across 6+ tenants with 600+ managed identities. The system coordinates sandboxed browser profiles, Porkbun/DreamHost APIs for automatic DNS TXT record verification, and scripted bulk user creation. A parallel factory runner handles multiple tenants at once, tracking progress through an Excel-based order system.

On the security side, I used Shodan to find vulnerable internet-facing systems including active crypto mining botnets, and responsibly disclosed findings to affected organisations. BigQuery pipelines I built process 1B+ records with deduplication, industry classification, and filter management. Infrastructure runs across Render, Fly.io, and Contabo with Docker, health checks, and automated rollback.

I also built a distributed cryptographic computation framework that splits sensitive algorithms across 4 geographically distributed VPS nodes using Shamir (3,4) threshold secret sharing. WireGuard mesh network with AES-256-GCM session keys, Ed25519 signature authentication, and nonce-based replay protection. You'd need to compromise 3 of 4 nodes (across Hetzner, DigitalOcean, Vultr, and Contabo) to reconstruct anything — information-theoretic security at $11/month.
